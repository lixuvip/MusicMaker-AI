# MusicMaker-AI v1.0.1 更新说明

更新时间：2026-05-02

`v1.0.1` 主要围绕项目化管理、历史筛选与整理效率、播放器快捷操作，以及 macOS / Windows 结构同步做了一轮整理，适合作为新的对外测试版本。

当前版本的生成能力仍以 MiniMax Music Generation API 为核心，这一版也保留了对 MiniMax 的明确致敬。正是因为现阶段接入路径足够稳定，MusicMaker-AI 才能优先把桌面端的生成、整理、试听和转码体验打磨起来。后续版本会在此基础上逐步增加对更多模型 API 的兼容支持。

## 版本亮点

### 1. 项目化管理

- 生成页新增“当前项目”面板
- 支持在生成前切换当前项目
- 支持新建项目与重命名项目
- 新生成的歌曲会先写入当前项目，再在项目下形成批次记录

这让同一轮创作、多轮筛选、不同主题项目之间的整理会更清晰。

### 2. 历史记录升级

- 历史记录支持按项目筛选
- 支持按轮次状态筛选
- 支持按最终选中状态筛选
- 支持按分类筛选
- 历史列表展示项目、批次、分类和筛选状态
- 详情页支持备注保存与清空
- 支持把单条记录移动到其他项目
- 支持推进下一轮、标记已选中、淘汰等整理动作

这次历史模块不再只是“看记录”，而是更接近一个可直接做筛选整理的工作台。

### 3. 展开区交互优化

- 详情页中的“展开详细参数 / 管理工具 / 音乐要求 / 歌词”已改成整条标题栏可点击
- 点击文字区域也可以展开或收起

相比之前只点左侧小图标，这一版更适合高频整理时快速操作。

### 4. 播放器与快捷键

- 底部全局播放器继续支持播放、暂停、拖拽播放和历史记录播放
- macOS 版补强了空格键播放 / 暂停快捷键
- 非文本输入状态下，空格可直接控制当前播放器

### 5. 跨平台结构继续对齐

- Windows 代码继续向 macOS 当前的信息结构和管理流程靠齐
- 为后续继续统一项目、历史、转码和播放器体验做准备

### 6. 模型接入方向说明

- 当前版本主要围绕 MiniMax 模型能力构建
- Release 文案与产品定位中会明确保留对 MiniMax 的致谢说明
- 后续版本会逐步扩展更多模型 API 的兼容层，而不是长期绑定单一接口

### 7. 下一阶段规划

- 声音克隆预计作为独立模块推进，不直接塞进当前音乐生成主界面
- 独立作词模块会优先考虑歌词撰写、润色、版本对比和复用体验
- 后续扩展会继续沿着“模块化工具箱”方向演进，避免把不同工作流都堆进一个页面里
- Windows 端会跟随 macOS 已验证能力继续补齐，保持功能命名和流程一致

## 当前版本保留能力

- 左侧模块切换：音乐生成 / 音频转码 / 历史记录
- MiniMax 音乐生成参数配置
- 单次生成 `1-10` 首，本地串行批量调度
- 分类在生成时即可确定，并自动写入历史
- 底部全局播放器
- 生成结果自动进入转码列表
- `ffmpeg` 检测与本地转码流程

## 本轮测试建议

建议优先测试以下场景：

1. 新建项目后生成 `2-3` 首音乐，确认批次归档是否正确
2. 在不同项目之间切换生成，检查历史筛选是否准确
3. 给历史条目添加备注、修改分类、移动到其他项目
4. 使用“推进下一轮 / 标记已选中 / 淘汰”整理一轮候选曲目
5. 测试展开区标题整行点击是否顺手
6. 在已有播放内容时，测试空格键暂停 / 继续播放
7. 检查转码模块和历史播放流程是否仍然正常
