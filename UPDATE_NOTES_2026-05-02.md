# MusicMaker-AI 更新说明

更新时间：2026-05-02

本次更新主要集中在 macOS 版本的使用体验、历史管理和批量生成能力，适合作为下一轮测试基线。

## 本次新增

### 1. 模块结构优化

- 左侧侧边栏现已拆分为三个独立模块：
  - 音乐生成
  - 音频转码
  - 历史记录

这样可以减少单页信息拥挤，方便在不同工作流之间切换。

### 2. 全局播放器

- 新增底部常驻播放器
- 支持播放 / 暂停
- 支持播放进度展示
- 支持拖拽本地音频文件到播放器区域直接播放
- 支持从历史记录中直接播放某一首
- 同一时间只播放一首歌
- 播放过程中仍可继续编辑生成参数、查看历史或执行转码

### 3. 音频转码增强

- macOS 版本增加 `ffmpeg` 自动检测
- 若未检测到 `ffmpeg`，界面会明确提示
- 支持通过 Homebrew 一键安装 `ffmpeg`
- 转码按钮会根据依赖状态自动启用或禁用

### 4. 批量生成

- 单次生成数量现支持选择 `1-10`
- 当前实现为本地串行批量生成
- 批量生成时会显示清晰的进度反馈：
  - 当前第几首
  - 已完成多少首
  - 总数量
- 批量生成时会自动禁用“生成完成后自动播放”

### 5. 收藏与分类

- 历史记录支持“收藏”
- 历史记录支持“分类”
- 收藏与分类是两套独立逻辑
- 分类可以在生成前直接填写，本次生成的所有歌曲都会自动继承
- 分类支持分类库复用
- 手动输入的新分类可加入分类库，后续直接选择
- 历史记录支持：
  - 仅看收藏
  - 按分类筛选

## 当前实现说明

- MiniMax 官方音乐生成接口当前按单首返回处理
- 因此“多首生成”目前采用本地连续调度，而非服务端单次返回多首
- Windows 版本仍在继续推进，当前仓库已保持统一目录结构和命名方式

## 本轮测试建议

建议优先验证以下场景：

1. 单首生成
2. 批量生成 `3-5` 首
3. 生成前填写分类并检查历史继承情况
4. 收藏、分类筛选是否正常
5. 转码模块中 `ffmpeg` 检测与安装提示
6. 全局播放器的播放、暂停、拖拽播放与历史播放

## 备注

本次更新暂未提交 Windows `.exe` 发布资产，后续可在 Windows 环境打包后补充上传至 Release。
